1. 스택

1-1. 스택이란 무엇인가

Stack의 영단어적 언어는 쌓여있는 더미 혹은 동사로, 쌓다를 의미합니다.
컴퓨터 공학에서도 이와 비슷합니다. 메모리 위에 차곡차곡 데이터를 쌓아가는 방식을 말합니다. stack에는 주요한 특징이 있습니다
LIFO(LAST IN FIRST OUT)구조: 나중에 들어온 것이 먼저 나갑니다.

1-2. 스택의 기능

  • append(var value): 스택의 맨뒤에 값을 추가합니다.
  • pop(): 스택의 맨뒤의 값을 삭제합니다.
  • peek(): 스택의 맨뒤의 값을 확인합니다.
배열 맨왼쪽이 배열의 시작입니다.

2. 큐

2-1. 큐란 무엇인가

QUEUE의 영단어적 의미는 무언가를 기다리는 줄을 의미합니다.
만약 음식점에서 차례를 기다리고 있다고 생각해봅시다. 그때 나보다 늦게 온 사람이 먼저 들어오면 화가 날 것입니다. 먼저 온 사람이 먼저 들어간다. 이게 큐의 핵심개념입니다.
FIFO(FIRST IN FIRST OUT)구조: 먼저 들어온 것이 먼저 나갑니다.

1-2. 큐의 기능

  • enqueue(var value): 큐 맨앞에 값을 추가합니다.
  • pop(): 큐 맨앞의 값을 삭제합니다.
  • peek(): 큐 맨앞의 값을 확인합니다.
배열 맨왼쪽이 배열의 시작입니다.
